# durable-streams
## Summary
Open HTTP-based protocol and toolkit for resumable, offset-based real-time streams to browsers, mobile, and native clients.

## Key Points
- Designed for catch-up reads + live tailing with durable replay semantics.
- Ships client libraries in many languages plus server, CLI, and conformance tests.
- Targets use cases like AI token streaming, client sync, and event-driven app updates.

## Related
- Docs and packages are organized under `packages/` and `examples/` in the monorepo
- Referenced as production-proven in Electric sync workflows
